# ML Research Tool for Audio Deepfake Detection

## 📋 Описание

**Инструмент для исследования методов машинного обучения при детектировании audio deepfake**

Это приложение предоставляет полнофункциональный интерфейс для:
- Извлечения акустических признаков из аудиофайлов
- Обучения моделей машинного обучения (Random Forest, XGBoost, KNN)
- Оптимизации гиперпараметров с использованием Optuna
- Оценки производительности моделей
- Экспорта результатов в Excel

## 🎯 Основные возможности

### 1. Извлечение признаков
- **Поддерживаемые признаки**:
  - RMS (Root Mean Square)
  - ZCR (Zero Crossing Rate)
  - Spectral Centroid (Спектральный центроид)
  - Spectral Rolloff (Спектральный спад)
  - Spectral Flatness (Спектральная плоскость)
  - MFCC (Mel-Frequency Cepstral Coefficients)
  - LFCC (Linear Frequency Cepstral Coefficients)
  - Spectral Contrast (Спектральный контраст)

- **Статистики**: mean (среднее), std (стандартное отклонение), skew (асимметрия), kurtosis (эксцесс)

### 2. Модели машинного обучения
- **Random Forest** - ансамбль деревьев решений
- **XGBoost** - градиентный бустинг
- **KNN** - метод k-ближайших соседей

### 3. Оптимизация гиперпараметров
- Использует **Optuna** для поиска оптимальных параметров
- Поддерживает настраиваемые диапазоны для каждого параметра
- Отображает прогресс оптимизации в реальном времени

### 4. Оценка производительности
- Метрики: Accuracy, Precision, Recall, F1-Score
- Равная вероятность ошибок (EER)
- Матрица ошибок
- Важность признаков (для Random Forest и XGBoost)
- График зависимости метрик от порога классификации

### 5. Экспорт результатов
- Сохранение результатов в формате Excel (.xlsx)
- Полная информация о конфигурации, параметрах и метриках
- Время выполнения каждого этапа

## 📖 Как использовать

### Запуск приложения
```bash
python gui_app.py
```

### Основной рабочий процесс

#### Шаг 1: Извлечение признаков
1. Укажите пути к папкам с аудиофайлами:
   - Папка обучения (Training Folder Path)
   - Папка валидации (Validation Folder Path)
   - Папка тестирования (Testing Folder Path)

2. Выберите признаки для извлечения (все выбраны по умолчанию)

3. Установите параметры извлечения:
   - **MFCC Coefficients**: количество MFCC коэффициентов (по умолчанию 20)
   - **LFCC Coefficients**: количество LFCC коэффициентов (по умолчанию 20)
   - **Statistics**: выберите статистики (mean, std, skew, kurtosis)

4. Нажмите **"Extract Features and Save CSV"**

#### Шаг 2: Выбор модели и гиперпараметров
1. Выберите модель из списка:
   - Random Forest
   - XGBoost
   - KNN

2. Установите диапазоны гиперпараметров:
   - Для Random Forest: n_estimators, max_depth, min_samples_leaf
   - Для XGBoost: n_estimators, learning_rate, max_depth, gamma, lambda, alpha, subsample, colsample_bytree
   - Для KNN: n_neighbors

3. Установите количество попыток оптимизации (Optuna Trials)

#### Шаг 3: Оптимизация гиперпараметров
1. Нажмите **"Run Hyperparameter Optimization"**
2. Дождитесь завершения оптимизации
3. Посмотрите найденные оптимальные параметры

#### Шаг 4: Финальная оценка
1. Отрегулируйте порог классификации если нужно (Selected Threshold)
2. Нажмите **"Run Final Evaluation"**
3. Просмотрите результаты и метрики

#### Шаг 5: Сохранение результатов
1. Нажмите **"Save Results to XLSX"**
2. Выберите место сохранения файла

### Альтернативный способ: Использование готовых CSV

Если у вас уже есть извлеченные признаки в формате CSV:

1. Перейдите в раздел **"Load Custom Feature CSVs (Overrides Extraction)"**
2. Укажите пути к файлам:
   - Custom Training CSV
   - Custom Validation CSV
   - Custom Testing CSV
3. Остальные этапы выполняются так же

## 📈 Понимание результатов

### Метрики оценки

- **Accuracy (Точность)**: доля правильно предсказанных образцов
  - Формула: (TP + TN) / (TP + TN + FP + FN)

- **Precision (Точность прогнозов для положительного класса)**: из предсказанных как положительные, сколько действительно положительные
  - Формула: TP / (TP + FP)

- **Recall (Полнота)**: из действительно положительных, сколько было найдено
  - Формула: TP / (TP + FN)

- **F1-Score**: гармоническое среднее Precision и Recall
  - Формула: 2 * (Precision * Recall) / (Precision + Recall)

- **EER (Equal Error Rate)**: порог, где FPR = FNR

- **Confusion Matrix (Матрица ошибок)**: таблица предсказаний vs действительных значений

Где:
- TP = True Positive (правильно предсказано положительное)
- TN = True Negative (правильно предсказано отрицательное)
- FP = False Positive (ошибочно предсказано положительное)
- FN = False Negative (ошибочно предсказано отрицательное)

### График "Metrics vs Threshold"

Показывает, как метрики изменяются в зависимости от порога классификации:
- **Выше кривая = лучше для этой метрики**
- **Точки пересечения = компромиссы между метриками**
- **Выбирайте порог на основе ваших требований**

## 🔧 Устранение проблем

### Проблема: "No module named 'feature_extractor'"
**Решение**: Убедитесь, что файл `feature_extractor.py` находится в той же папке, что и `gui_app.py`

### Проблема: Приложение долго извлекает признаки
**Решение**: 
- Это нормально для больших наборов данных
- Процесс можно отменить, если закрыть приложение
- Процесс работает в отдельном потоке и не блокирует UI

### Проблема: "Invalid range for parameter"
**Решение**: Убедитесь, что введены только числовые значения в поля диапазонов гиперпараметров

### Проблема: CSV файлы не найдены после извлечения
**Решение**: 
- Проверьте, успешно ли завершилось извлечение признаков
- Убедитесь, что у вас есть права на запись в папку проекта
- Попробуйте вручную указать пути в разделе "Load Custom Feature CSVs"

### Проблема: Недостаточно памяти
**Решение**:
- Используйте меньший набор данных
- Закройте другие приложения
- Установите более высокое значение max_depth или используйте KNN (медленнее, но менее требователен к памяти)

### Проблема: Модель плохо работает
**Решение**:
- Попробуйте другую модель
- Расширьте диапазоны гиперпараметров
- Увеличьте количество Optuna Trials
- Проверьте качество данных
- Попробуйте другие признаки

## 💡 Советы по использованию

### Для начинающих
1. Начните с Random Forest (более понятная модель)
2. Используйте рекомендуемые диапазоны гиперпараметров
3. Запустите с меньшим количеством trials (10-20) для тестирования
4. Постепенно увеличивайте trials по мере улучшения результатов

### Для опытных пользователей
1. Экспериментируйте с XGBoost для лучших результатов
2. Используйте разные комбинации признаков
3. Сравнивайте результаты разных моделей
4. Анализируйте важность признаков
5. Используйте EER порог для сбалансированной работы

### Рекомендуемые комбинации

**Высокая точность:**
- Модель: XGBoost
- Trials: 100+
- n_estimators: 100-200
- learning_rate: 0.01-0.1

**Быстрая работа:**
- Модель: KNN
- n_neighbors: 5-10
- Меньше признаков

**Сбалансированная:**
- Модель: Random Forest
- Trials: 50
- Порог: EER

## 🔗 Дополнительная информация

### Форматы данных

**Входные CSV файлы должны иметь колонки:**
- `filename` - имя аудиофайла
- `label` - метка (0 или 1): 0 = оригинальный, 1 = deepfake
- `duration` - длительность аудио в секундах
- Остальные колонки - извлеченные признаки

**Пример:**
```
filename,label,duration,rms_mean,rms_std,zcr_mean,...
audio_001.wav,0,2.5,0.45,0.12,0.08,...
audio_002.wav,1,2.3,0.52,0.15,0.11,...
```
